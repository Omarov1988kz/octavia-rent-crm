# Локальный запуск Octavia Rent CRM

Обычный запуск CRM на Windows не требует VS Code.

## Запуск CRM

1. На этом компьютере проект должен лежать строго в папке:
   `C:\projects\rentcrm2`
2. Откройте папку проекта `C:\projects\rentcrm2`, где лежит файл `start-octavia-crm.bat`.
3. Дважды нажмите `start-octavia-crm.bat`.
4. CRM запустится на:
   `http://localhost:3002/admin/bookings`

Скрипт сам поднимает SSH-туннель к базе данных:

`ssh -L 5432:127.0.0.1:5432 root@31.129.102.131`

Пароль вводить не нужно, потому что для входа на VPS используется SSH-ключ.

Во время работы CRM не закрывайте окна `CRM DB Tunnel` и `Octavia Rent CRM Server`. Если закрыть окно туннеля, CRM потеряет соединение с базой. Если закрыть окно сервера, CRM перестанет открываться в браузере.

Сейчас стартовый скрипт временно запускает CRM в dev-режиме командой:

`npm run dev -- -p 3002`

## Если данные не загружаются

Если автомобили, брони или другие данные не загружаются, проверьте, что окно `CRM DB Tunnel` открыто и не показывает ошибку подключения.

Также проверьте:

- файл `.env.local` настроен;
- порт `5432` не занят другой программой;
- порт `3002` не занят другой CRM или Next.js сервером.

## Обновление CRM

Для обновления запустите `update-octavia-crm.bat` из папки проекта `C:\projects\rentcrm2`.

Скрипт выполнит:

1. `git pull`
2. `npm install`
3. `npm run build:local`

После успешного обновления появится сообщение:

`Обновление завершено. Теперь можно запустить start-octavia-crm.bat`

Если на одном из шагов будет ошибка, обновление остановится и покажет сообщение.

## Важные условия

- Для обычного запуска VS Code не нужен.
- На этом компьютере путь к папке проекта важен: используйте строго `C:\projects\rentcrm2`.
- Публичный сайт и календарь этим запуском не меняются.
- Деплой этим запуском не выполняется.
